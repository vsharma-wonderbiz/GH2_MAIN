### 4️⃣ Fetch TagIds from Tag table

Using those names:

```
pressure → 11
outlet_pressure → 12
```

Now you have dependent TagIds.

---

### 5️⃣ Fetch MappingIds (VERY IMPORTANT STEP)

Now combine:

* AssetIds (all stacks)
* Dependent TagIds

Query mapping table:

```
MappingId | AssetId | TagId
--------------------------------
101       | 1       | 11
102       | 1       | 12
201       | 2       | 11
202       | 2       | 12
```

Now you have correct mapping ids per stack.

---

### 6️⃣ Fetch average sensor data using MappingIds

Since your avg table stores data using MappingId:

```
SELECT * FROM AvgSensorData
WHERE MappingId IN (101,102,201,202)
```

Now you have values.

---

### 7️⃣ Group data per Asset

Build something like:

```
Stack_1:
   pressure = 25
   outlet_pressure = 22

Stack_2:
   pressure = 26
   outlet_pressure = 23
```

---

### 8️⃣ Calculate KPI per asset

For example:

```
pressure_diff = pressure - outlet_pressure
```

So:

```

Stack_2 → 3
```

### 9️⃣ Send clean response for graphs
